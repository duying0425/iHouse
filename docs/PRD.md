# iHouse · 居所图鉴 — 产品需求文档（PRD）

> 版本：v2 · 最后更新：2026-07-01
> 一句话定位：把家当成一座小型博物馆，为每件物品/设施建立一份带图的、可定位的、可打印归档的电子档案。

## 1. 背景与原始设想

日常生活中会遇到这些场景：

- 「这台净水器的滤芯型号是什么？什么时候买的？」翻遍抽屉找不到说明书。
- 「上次那个空调维修师傅问铭牌上的型号，在哪台机器上？」记不清。
- 「家里东西越来越多，到底哪些区域塞满了、哪些还有富余？」心里没数。
- 「想给爸妈一份家里所有设施清单，方便他们维修时参考。」没有现成资料。

原始设想是做一份**「居所图鉴」**——像图鉴一样，把家拆解为「户型 → 区域 → 物品」三层，每个物品都有照片、位置、参数、备注，最终能导出成一份可打印的 PDF 手册，平时手机/电脑随时查、需要时打印归档。

## 2. 目标用户与场景

| 用户场景 | 使用方式 |
| --- | --- |
| 个人/家庭主人 | 手机随手拍照录入物品，电脑大屏整理归类，定期导出 PDF 备份 |
| 多设备用户 | 电脑录入、手机查阅，同一份数据多端共享 |
| 搬家/装修 | 导出 PDF 作为物品清单交给搬家公司或装修队 |
| 售后/维修 | 检索关键词秒查型号、购买日期、价格、备注 |
| NAS 自部署用户 | 在自家 NAS 上跑一份，全家访问，数据不外流 |

## 3. 核心功能

### 3.1 户型图管理
- 导入自己的户型图（支持上传/粘贴，自动压缩）
- 在户型图上为每个区域设置锚点位置
- 可拖拽调整锚点，区域在户型图上有视觉边界

### 3.2 区域划分
- 预置常见区域：入户玄关、主卧、次卧、客厅、餐厅、厨房、卫生间、阳台等
- 每个区域可自定义名称、描述
- **每个区域支持 1 张或多张图**（区域总图、设施图、某面墙等），可编辑标签，多选上传保持顺序

### 3.3 物品/设施录入
- 录入字段：名称、分类、品牌、规格、购买日期、价格、备注、主图、多张附属图（gallery）
- 分类：家电 / 家具 / 储物 / 装饰 / 管线设施 / 其他，每类有展示色
- **位置标注**：录入时直接在区域图上点选位置，红色标记点直观展示
- 支持粘贴上传（截图直接 Ctrl+V），自动压缩
- 一个物品关联到某张区域图上的某个坐标

### 3.4 检索
- 按关键词（名称/品牌/规格/备注/区域名模糊匹配）
- 按分类、品牌、区域筛选
- 按名称 / 购买日期 / 价格排序

### 3.5 PDF 导出（图鉴生成）
按层级生成图鉴，页序为：**封面 → 户型图 → 各区域 → 各物品**
- 封面页：标题、副标题、统计概览
- 户型图页：整屋户型图 + 所有区域锚点
- 区域页：区域图 + **图上标注所有设备位置序号** + 物品清单表（序号对照）
- 物品页：物品主图 + 详情字段

点击「打印 / 导出 PDF」唤起浏览器原生打印对话框，在「目标」选「另存为 PDF」即可保存。浏览器原生渲染，秒级完成，文字矢量清晰。

### 3.6 多设备数据共享
- 数据存服务器 SQLite，所有设备访问同一份数据
- 浏览器 IndexedDB 作本地缓存，离线仍可用，联网自动同步
- 600ms 防抖写入，页面隐藏时 `keepalive` 尽力同步

### 3.7 数据维护
- 一键导出 JSON 备份（跨设备/跨环境迁移）
- 导入 JSON 备份恢复
- 旧版本数据自动迁移（v1 → v2 schema）

## 4. 数据模型

```
Home
├── title: string            // 居所名称
├── subtitle?: string        // 副标题
├── floorPlanImage: string   // 户型图（base64）
└── areas: Area[]
      ├── id: string
      ├── name: string
      ├── floorPlanPos: {x, y}        // 在户型图上的锚点（百分比 0-100）
      ├── bounds?: {x, y, w, h}       // 户型图上的高亮矩形
      ├── description?: string
      ├── images: AreaImage[]         // 区域图片（1 张或多张）
      │     ├── id: string
      │     ├── url: string           // base64
      │     └── label?: string        // 如：总图、设施图、东墙
      └── items: Item[]
            ├── id: string
            ├── areaId: string
            ├── name: string
            ├── category: 家电|家具|储物|装饰|管线设施|其他
            ├── brand?: string
            ├── spec?: string
            ├── purchaseDate?: string  // YYYY-MM-DD
            ├── price?: number
            ├── remark?: string
            ├── image: string          // 主图 base64
            ├── gallery?: string[]     // 附属图
            ├── areaImageId?: string   // 关联到 AreaImage.id
            └── areaImagePos?: {x, y}  // 在该区域图上的位置（百分比）
```

存储方式：整个 Home 对象序列化为 JSON，存入 SQLite 单行（`id=1`），整体覆盖写入。

## 5. 非功能性需求

- **图片压缩**：上传/粘贴时自动 canvas 缩放 + JPEG 压缩（户型图 ≤2000px、区域图 ≤1600px、物品图 ≤1200px，质量 0.82-0.85），避免存储爆满
- **响应式**：桌面优先，移动端可用
- **视觉风格**：暖色调（cream/ink/clay），衬线标题 + 无衬线正文，纸质感图鉴风格
- **部署**：支持 Docker / Synology NAS Container Manager 部署
- **数据安全**：自部署，数据不出局域网

## 6. 边界与不做的事

- **不做**：多用户账号系统（单用户/家庭自用，多设备共享同一份数据）
- **不做**：复杂冲突合并（last-write-wins，单用户场景够用）
- **不做**：云端托管服务（只提供自部署方案）
- **不做**：条码/二维码扫描录入（可后续按需加）
- **不做**：物品保质期/维护周期提醒（聚焦档案而非提醒）
